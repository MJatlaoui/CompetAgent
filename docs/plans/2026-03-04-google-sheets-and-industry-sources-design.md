# Google Sheets Persistence + Industry Sources Design

**Date:** 2026-03-04
**Status:** Approved

---

## Goal

Two combined changes to the CompetAgent pipeline:
1. Replace Notion battlecard persistence with Google Sheets
2. Add 100 industry publication sources with a validation-before-production workflow

---

## Part 1 — Google Sheets Persistence

### Problem

Notion is unavailable. The `write_to_battlecard()` function in `src/persistence.py` and the `approve_check.yml` workflow need a new storage backend.

### Design

**Authentication:** Google service account JSON stored as `GOOGLE_CREDENTIALS_JSON` GitHub Secret. Loaded at runtime via `json.loads(os.environ["GOOGLE_CREDENTIALS_JSON"])` — no temp files, no base64.

**Library:** `gspread>=6.0.0` + `google-auth>=2.0.0`

**Sheet structure:** One header row (auto-written on first run if sheet is empty), then one data row per approved insight.

| Column | Field | Type |
|--------|-------|------|
| A | Headline | text |
| B | Competitor | text |
| C | Type | text |
| D | Score | number |
| E | Source URL | text |
| F | Date Added | date (ISO) |
| G | Sales Angle | text |
| H | Gap Analysis | text |
| I | Priorities | comma-separated text |

**Credentials flow:**
```
GOOGLE_CREDENTIALS_JSON (secret)
  → json.loads()
  → Credentials.from_service_account_info()
  → gspread.authorize()
  → sheet.append_row([...])
```

### Files Changed

| File | Change |
|------|--------|
| `src/persistence.py` | Rewrite — swap notion_client for gspread |
| `requirements.txt` | Remove `notion-client`, add `gspread>=6.0.0`, `google-auth>=2.0.0` |
| `.github/workflows/approve_check.yml` | Swap `NOTION_*` env vars for `GOOGLE_CREDENTIALS_JSON` + `GOOGLE_SHEET_ID` |
| `tests/test_persistence.py` | Update mock — assert `ws.append_row()` called with correct values |

### Files Unchanged

`src/approve.py` — calls `write_to_battlecard(insight)` with no knowledge of the backend.

## Part 2 — Industry Sources

### Problem

The pipeline currently monitors 6 competitors only. 100 industry publications need to be added, but each URL must be technically validated (HTTP 200, valid RSS, ≥1 entry) before going live.

### Design

**Source separation:** Competitor feeds stay in `config/sources.yaml`. Industry feeds live in `config/industry_sources.yaml` (auto-generated by the validation script). `load_sources()` reads both files.

**Pending source format** (`config/pending_sources.json`):
```json
[
  {
    "name": "CX Today",
    "category": "CX / CCaaS News",
    "url": "https://www.cxtoday.com/feed/",
    "type": "rss",
    "notes": "High-signal CX news"
  }
]
```

**Validation rules (A — technical only):**
- HTTP 200 response within 15s timeout
- Response parses as valid RSS/Atom XML
- At least 1 entry present

**Special cases:**
- Sources with `"url": null` or notes containing "No native RSS" / "No official RSS" → auto-SKIP (not a failure)
- Sources with `"type": "html"` → skipped in this pass (HTML validation is more complex, deferred)

**Validation output:**
- Printed table to stdout: PASS / FAIL / SKIP with reason
- `config/validation_report.json` committed to repo as a permanent record
- All PASS sources appended to `config/industry_sources.yaml` (idempotent — no duplicates)

**GitHub Actions workflow** (`validate_sources.yml`):
- Manual trigger only (`workflow_dispatch`)
- Runs `python tools/validate_sources.py config/pending_sources.json`
- Commits updated `config/industry_sources.yaml` and `config/validation_report.json`

### Files Added

| File | Purpose |
|------|---------|
| `config/pending_sources.json` | All 100 sources to validate |
| `config/industry_sources.yaml` | Validated sources (auto-generated, committed) |
| `tools/validate_sources.py` | Validation script |
| `tools/__init__.py` | Package marker |
| `.github/workflows/validate_sources.yml` | Manual-trigger validation workflow |

### Files Changed

| File | Change |
|------|--------|
| `src/sources/__init__.py` | `load_sources()` reads both `sources.yaml` and `industry_sources.yaml` |
| `tests/test_sources.py` | Add test for multi-file loading |
